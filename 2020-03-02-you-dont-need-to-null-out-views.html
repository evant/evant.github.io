linkYou Don't Need to Null Out Views

There's been a long standing footgun when using fragments on Android because the fragment may live longer than it's views. Solutions to this have ranged from ignoring the problem to some complicated rxjava setup. However, with the additions to AndroidX this is no longer necessary!

The trick is to scope all view interactions to onViewCreated(). 

1class MyFragment : Fragment(R.layout.my_fragment) {

2  override fun onViewCreated(view: View, savedInstanceState: Bundle?) {

3    val binding = MyBinding.bind(view)

4    binding.text = "Look, no leaks!"

5  }

6}

If you don't assign any views to fragment fields, you don't need to worry about clearing them out.

Sidenote: if you are not familiar with some of the features here, check out the new fragment constructor and view binding.

Now, you may be wondering how you are supposed to accomplish much with this limited scope, but with a few other AndroidX features it turns out you can do quite a lot.

linkListening for updates

The other key part to this is viewLifecycleOwner. This gives you a scope that you can use for asynchronous operations. For example, you can listen to liveData:

1override fun onViewCreated(view: View, savedInstanceState: Bundle?) {

2  val binding = MyBinding.bind(view)

3  viewModel.title.observe(viewLifecycleOwner) { text -> binding.text = text }

4}

Or scope a coroutine/Flow operation:

1override fun onViewCreated(view: View, savedInstanceState: Bundle?) {

2  val binding = MyBinding.bind(view)

3  viewLifecycleOwner.lifecycleScope.launchWhenStarted {

4    viewModel.title.collect { text -> binding.text = text } 

5  }

6}

linkLifecycle Events

If you need to handle view-related things in other lifecycle events like onPause/onResume, you can attach your own LifecycleObserver.

1override fun onViewCreated(view: View, savedInstanceState: Bundle?) {

2  val binding = MyBinding.bind(view)

3  viewLifecycleOwner.lifecycle.addObserver(object: DefaultLifecycleObserver {

4    override fun onResume(owner: LifecycleOwner) {

5      Snackbar.make(view, "OnResume Called", Snackbar.LENGTH_LONG).show()

6    }

7  })

8}

linkOther Callbacks

The one place where this gets more tricky is if you need to handle other callbacks that haven't yet been broken out by AndroidX. For example, runtime permissions. In these cases, I would recommend routing the event in a way you can react to it.

For example, you could use an event wrapper with live data:

1private val showCamera = MutableLiveData<Event<Boolean>>()

2

3override fun onViewCreated(view: View, savedInstanceState: Bundle?) {

4  val binding = MyBinding.bind(view)

5

6  showCamera.observe(viewLivecycleOwner) { event ->

7    event.getContentIfNotHandled()?.let { permission ->

8      if (permission) {

9        startActivity(Intent(requireContext(), CameraActivity::class.java)

10      }

11    }

12  }

13

14  binding.showCameraButton.setOnClickListener {

15    if (ContextCompat.checkSelfPermission(

16        requireContext(), 

17        Manifest.permission.CAMERA == PackageManager.PERMISSION_GRANTED

18    ) {

19      showCamera.value = Event(true)

20    } else {

21      requestPermissions(arrayOf(Manifest.permission.CAMERA), 0)

22    }

23  }

24}

25

26override fun onRequestPermissionsResult(

27  requestCode: Int,

28  permissions: Array<out String>,

29  grantResults: IntArray

30) {

31  if (requestCode == 0 && grantResults.size == 1) {

32    showCamera.value = Event(grantResults[0] == PackageManager.PERMISSION_GRANTED)

33  }

34}

Or use a Channel:

1private val showCamera = Channel<Boolean>(1)

2

3override fun onViewCreated(view: View, savedInstanceState: Bundle?) {

4  val binding = MyBinding.bind(view)

5

6  viewLifecycleOwner.lifecycleScope.launchWhenStarted {

7    for (permission in showCamera) {

8      if (permission) {

9        startActivity(Intent(requireContext(), CameraActivity::class.java)

10      }

11    }

12  }

13

14  binding.cameraButton.setOnClickListener {

15    if (ContextCompat.checkSelfPermission(

16        requireContext(), 

17        Manifest.permission.CAMERA == PackageManager.PERMISSION_GRANTED

18    ) {

19      showCamera.offer(true)

20    } else {

21      requestPermissions(arrayOf(Manifest.permission.CAMERA), 0)

22    }

23  }

24}

25

26override fun onRequestPermissionsResult(

27  requestCode: Int,

28  permissions: Array<out String>,

29  grantResults: IntArray

30) {

31  if (requestCode == 0 && grantResults.size == 1) {

32    showCamera.offer(grantResults[0] == PackageManager.PERMISSION_GRANTED)

33  }

34}

By scoping your views to onViewCreated you don't have to worry about nulling them out our calling them at the wrong time. And AndroidX gives you the tools to do it!
